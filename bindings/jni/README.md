# pls4all — JNI (JVM) binding

Desktop JNI binding for libp4a. Exposes the SIMPLS PLS regression
through `io.github.pls4all.Pls4all.plsFit(...)`. Parity-gated
against the shared cross-binding fixture
(`bindings/js/test/parity_fixture.json`) at `rmse_rel = 0.0` — i.e.
bit-exact with the native Python reference (Java and the C ABI both
use IEEE 754 doubles and share the same row-major buffer layout, so
no transpose is needed).

Android packaging is on the backlog (Phase 36): the same Java class
and JNI C source compile against the Android NDK, plus an AAR wrapper
around the resulting `libp4a.so + libp4a_jni.so`.

## Layout

```
bindings/jni/
├── java/io/github/pls4all/Pls4all.java   Public Java API
├── c/p4a_jni.c                            JNI <-> p4a_pls_fit_simple bridge
├── c/io_github_pls4all_Pls4all.h          Generated by javac -h
├── test/TestParity.java                   Parity gate vs shared fixture
├── build.sh                               One-shot build script
└── build/                                 Output (.so, .class) — gitignored
```

## Build

```bash
JAVA_HOME=/path/to/jdk-17 bindings/jni/build.sh
```

The script auto-detects `JAVA_HOME` from the system `javac` if not
set. On conda-forge openjdk the JDK lives under
`<env>/lib/jvm`; the script handles that automatically.

## Parity gate

```bash
LD_LIBRARY_PATH=$(pwd)/build/dev-release/cpp/src \
PLS4ALL_JNI_LIB=$(pwd)/bindings/jni/build/libp4a_jni.so \
$JAVA_HOME/bin/java -Drepo.root=$(pwd) \
  -cp $(pwd)/bindings/jni/build/classes TestParity
```

## Usage

```java
import io.github.pls4all.Pls4all;

double[] x = ...;          // length n * p, row-major
double[] y = ...;          // length n * q, row-major
int n = 50, p = 5, q = 1, k = 3;

Pls4all.FitResult fit = Pls4all.plsFit(x, y, n, p, q, k);
System.out.println(java.util.Arrays.toString(fit.coefficients));
// length p*q
```

The library is loaded from `System.loadLibrary("p4a_jni")` (i.e. the
JVM's `-Djava.library.path`) by default. To override, set the
`PLS4ALL_JNI_LIB` env var to the absolute path of `libp4a_jni.so`
before the JVM starts.

Both `libp4a.so` and `libp4a_jni.so` must be reachable at load time;
the easiest way during development is `LD_LIBRARY_PATH` pointing at
the CMake `dev-release/cpp/src` directory.
